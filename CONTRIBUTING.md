# Contributing Guide

Thanks for contributing.

## Rules

- Keep things practical
- No unnecessary theory
- Code must be runnable
- Keep dependencies minimal

## How to Contribute

1. Fork the repo
2. Create a branch
3. Add your improvement
4. Submit PR

## Good Contributions

- New attack examples
- Better filters
- Real-world configs
- Security fixes

## Bad Contributions

- Overly academic content
- Unverified claims
- Broken code

---

Security is a moving target. Help keep this repo sharp.
