# Monitoring and Logging

## Why This Matters

Attacks don’t announce themselves.

You detect them via:
- Logs
- Metrics
- Alerts

---

## What to Log

- Requests
- Errors
- Blocked inputs
- Rate limit hits

---

## What NOT to Log

- API keys
- Passwords
- Sensitive user data

---

## Goal

Detect:
- Prompt injection attempts
- Abuse patterns
- Data leaks

---

## Rule

> If you can’t see it, you can’t stop it.
